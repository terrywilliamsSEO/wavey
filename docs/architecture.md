# Architecture Notes

Last updated: 2026-06-21

## Entry Point

`main.py` defines the CLI. Current important commands:

- `run`: one simulation.
- `sweep`: parameter sweep.
- `diagnose-run`: mode-shape diagnostics for one run.
- `artifact-controls`: targeted sponge-boundary controls.
- `dt-control`: targeted smaller-time-step controls.
- `grid-control`: larger matched-proportion grid control.
- `fixed-domain-grid-control`: true same-domain grid-refinement control.
- `resolution-diagnostics`: fixed-domain source, mask, energy-budget, radial-profile, and mode-shape resolution audit.
- `source-normalized-resolution-diagnostics`: source-normalized fixed-domain resolution audit with legacy `per_cell` reference variants.
- `breathing-period-audit`: read-only peak-picking audit for completed diagnostic runs.
- `core-modal-probe`: controlled source-normalized fixed-domain boundary references plus direct core excitation probes.
- `transport-controls`: targeted matched-work boundary-geometry and annulus/near-defect source controls.
- `prototype-3d`: tiny fixed-domain 31^3 shell-breathing prototype for the 2D boundary-flux mechanism.
- `prototype-3d-audit`: read-only failure-mode audit for completed tiny 3D prototype runs.
- `prototype-3d-source-sponge-control`: tiny 31^3 source/sponge separation control for the 3D boundary source.
- `prototype-3d-sponge-strength-control`: tiny 31^3 sponge-strength/width control for the inner-sponge-edge 3D boundary source.
- `prototype-3d-source-geometry-control`: tiny 31^3 source-geometry comparison from the stronger-sponge inner-edge 3D setup.
- `prototype-3d-cubic-focus-control`: tiny 31^3 focused control around the clean six-face cubic 3D boundary source.
- `prototype-3d-cubic-confirmation-control`: tiny 31^3 dt/sponge confirmation around the original and sign-flipped cubic 3D boundary phases.
- `prototype-3d-grid-confirmation-control`: tiny fixed-domain 31^3 to 41^3 grid confirmation for the clean sign-flipped cubic 3D source.
- `prototype-3d-threshold-control`: tiny calibrated 41^3 amplitude/phase tolerance check for the clean sign-flipped cubic stronger-sponge 3D source.
- `prototype-3d-defect-control`: tiny calibrated 41^3 defect-ablation control for the clean sign-flipped cubic stronger-sponge 3D source.
- `prototype-3d-radial-window-audit`: tiny 41^3 current-defect vs neutral-lattice radial shell-window audit for defect-lift measurement.
- `prototype-3d-defect-lift-sweep`: tiny 41^3 stronger/different-defect sweep against the neutral-lattice shell-tail baseline.
- `prototype-3d-interference-diagnostics`: tiny 41^3 neutral-lattice phase/interference diagnostic for the structured boundary-transport hypothesis.
- `prototype-3d-standing-persistence`: tiny 41^3 settled shell-pattern persistence check for the two clean neutral cubic variants.
- `prototype-3d-transport-packet-audit`: tiny 41^3 motion/flux audit for the clean neutral cubic shell-window tail.
- `prototype-3d-packet-lifecycle-audit`: extended tiny 41^3 lifecycle audit for whether the clean cubic packet exits, diffuses, stalls, or repeatedly refocuses.
- `prototype-3d-refocusing-engineering-control`: tiny 41^3 source-shaping control that varies phase offset, cutoff timing, frequency, and optional chirp around the clean neutral cubic packet.
- `prototype-3d-refocusing-map-control`: tiny 41^3 cutoff-frequency cross-map around the refocusing winners.
- `prototype-3d-cutoff-phase-map-control`: tiny 41^3 release-phase timing map around cutoff 18, with compact phase-offset and sign-flip/polarity families, passive island-refinement, phase-lock needle, and threshold-robust confirmation presets, ranked cutoff-phase output, release-phase island stability checks, phase-lock needle-width reporting, event-threshold sensitivity auditing, and conservative threshold-robust scoring.
- `prototype-3d-resonator-layer-control`: tiny 41^3 passive boundary-inner-edge resonator layer mechanism test around the narrow sign-flip phase-lock cluster, with threshold-robust scoring, phase-lock cluster-width checks, resonator/lattice/coupling energy accounting, and contamination auditing.
- `prototype-3d-release-phase-return-map`: read-only 3D release-phase predictor that consumes existing cutoff/refocusing run artifacts, builds per-row phase and early-packet features, fits simple interpretable predictors, and recommends five blind-confirmation cutoffs without running new physics.
- `prototype-3d-release-phase-blind-confirmation`: fixed five-cutoff 41^3 passive confirmation of the return-map phase rule, with prediction-role comparison, strict 0.35/0.40 counts, threshold-free metrics, and no post-result tuning.
- `prototype-3d-release-phase-numerical-validation`: fixed four-cutoff baseline/half-dt validation of the blind-confirmed phase rule, with optional explicit quarter-dt rows, strict 0.35/0.40 counts, threshold-free metrics, and dt-comparison output.
- `prototype-3d-release-phase-dt-recenter`: fixed half-dt-only 41^3 recentering map around the surviving release-phase row, testing only the preselected 17.930-17.950 cutoffs plus low-side controls and writing summary plus threshold-robust score output.
- `prototype-3d-release-phase-proof-pack`: fixed quarter-dt 41^3 proof pack for the recentered release-phase pocket, with a frozen candidate card, predeclared gates, threshold-free lifecycle metrics, strict 0.35/0.40 counts, and no user-facing cutoff sweep knob.
- `prototype-3d-release-phase-resolution-lift`: one-step quarter-dt resolution lift for the passive release-phase rule, defaulting to a single recalibrated `51^3` candidate plus two fixed phase controls, with matched work, threshold-robust scoring, lifecycle gates, and energy-accounting output.
- `prototype-3d-release-phase-resolution-postmortem`: read-only comparison of the `41^3` proof-pack winners against the failed `51^3` candidate and controls, with event-threshold shrinkage, timing, radial alignment, packet width/spread, return phase, and recalibration-prediction outputs.
- `prototype-3d-release-phase-modal-audit`: read-only modal comparison of the `41^3` proof cluster, failed `51^3` lift rows, resolution postmortem, and central HF burst contrast, with shell spectra, bandwidth/concentration, autocorrelation decay, return jitter, peak widths, radial packet spread, scalar phase-locking proxy, and loose-vs-strict count shrinkage.
- `prototype-3d-release-phase-dispersion-audit`: read-only dispersion/blur model for the `41^3` proof rows versus failed `51^3` lift rows, with deterministic source/shell geometry reconstruction, modal-band checks, bandwidth and tail-radius consistency, source-discretization checks, shell-window sampling checks, and spatial-phase-frame availability gates.
- `prototype-3d-smooth-envelope-resolution-lift`: fixed three-row `51^3` source-spectrum rescue test after the source-spectrum audit, comparing hard cutoff, same-release smooth `sin^2` candidate, and smooth weak-side control with frequency, phase, work, grid, lattice, sponge, source geometry, and shell window fixed.
- `prototype-3d-boundary-phase-conjugate-control`: fixed measured patch-level boundary wavefront test after spatial phase/source-spectrum scale checks fail, deriving a coarse frozen 51^3 phase/amplitude mask from 41^3 proof shell phase frames and comparing it with hard, shuffled-patch, amplitude-only, phase-only, and wrong-return controls.
- `prototype-3d-modal-sparsity-audit`: read-only modal sparsity audit over existing proof/lift/spatial/smooth/phase-conjugate artifacts, measuring return timing regularity, sparse sinusoid reconstruction counts, modal participation ratio, peak width versus modal density, and whether hard/smooth/phase-conjugate/shuffled `51^3` controls share one signature.
- `prototype-3d-return-family-gate-audit`: read-only detector/mechanism audit over existing proof/lift/spatial/smooth/phase-conjugate/modal-sparsity artifacts, measuring predicted return-window occupancy, comb score, indexed return strength, threshold crossings, off-comb energy, and late-return survival to separate gate artifacts from real return-family weakening.
- `prototype-3d-off-comb-leakage-audit`: read-only failure-localization audit over the same artifacts plus the return-family gate result, comparing radial leakage, angular-sector coherence, delayed outer/off-comb correlation, modal sideband leakage, spatial-pattern leakage, and flux balance.
- `prototype-3d-return-pattern-symmetry-audit`: read-only spatial-pattern identity audit over saved proof/lift/spatial/smooth/phase-conjugate/return-family/off-comb artifacts, testing whether sign, global phase, cubic rotations/reflections, angular sector shifts, penalized sector permutations, or harmonic coefficients rescue `51^3` return-pattern memory.
- `prototype-3d-branch-closure-report`: read-only closure packet over the full passive scale-lift evidence chain, freezing the `41^3` claim, `51^3` non-claim, failure explanation, forbidden paths, and reopen criteria without running physics.
- `prototype-3d-spatial-memory-mechanism-lab`: independent passive mechanism lab for return-to-return spatial-pattern memory, starting with a fixed small `41^3` mechanism set and running the optional `51^3` follow-up only if a `41^3` mechanism beats neutral and randomized controls under clean gates.
- `prototype-3d-cubic-memory-tradeoff-map`: fixed `41^3` local strength/orientation map for cubic degeneracy splitting, testing whether the spatial-memory advantage can be preserved without dropping below strict `9/8`, with matched randomized controls and no default `51^3` path. The first run classified as `memory_only_tradeoff_supported`, so it documents a clean memory/count tradeoff rather than a strict-count-preserving setting.
- `prototype-3d-cubic-memory-survivor-bias-audit`: read-only audit over the cubic-memory tradeoff artifacts, comparing surviving-peak, equal first-N return-index, and neutral-predicted-window memory to separate real same-window gain from survivor-window inflation.
- `prototype-3d-isochronous-cubic-memory-anchor`: fixed `41^3` mechanism test combining cubic degeneracy splitting with a smooth radial compensation profile to test whether memory can improve while strict count, comb score, and off-comb energy stay near neutral. The first run classified as `memory_only_anchor_tradeoff`.
- `prototype-3d-isochronous-anchor-cleanup-control`: fixed `41^3` cleanup control for the isochronous anchor's off-comb penalty, comparing the current `0.5x` anchor reference with smooth-taper cleanup variants and a matched randomized control. The first run classified as `cleanup_memory_only_tradeoff`.
- `prototype-3d-angular-mode-cleanup-control`: fixed `41^3` passive angular high-mode damping control for the isochronous anchor's off-comb penalty, comparing the current `0.5x` anchor reference with weak/medium/cubic-preserving angular cleanup rows and a randomized matched damping control. Implemented but not yet run.
- `prototype-3d-central-burst-control`: firewalled `central_hf_scattering_branch` command for neutral `41^3` central tiny-radius velocity bursts with no boundary drive, no active second pulse, no resonator, no defect variants, a fixed frequency/energy ladder, event-threshold counts, energy accounting, packet/flux diagnostics, and one automatic half-dt check of the best baseline row.
- `prototype-3d-second-pulse-control`: tiny 41^3 active second-pulse control from the best release phase, with no-pulse and passive-extension work-normalized comparators, optional multi-scale/multi-duration reduced-work maps, timing audit output, and travel-time-adjusted micro-map mode.

## Core Modules

- `simulation/config.py`: dataclasses for simulation, sweep, defect, driver config, and direct core-drive options. Also owns fixed-domain helpers: `nx`, `ny`, `dx`, `dy`, physical radii, cell area, and shape.
- `simulation/lattice.py`: 2D oscillator lattice, masks, sponge damping, coupling force, semi-implicit Euler step, energy density, and separate boundary/core external force paths.
- `simulation/drivers.py`: boundary emitter coverage weights, source normalization, masks, phase maps, and direct core/annulus drive masks.
- `simulation/metrics.py`: per-step metrics and post-hoc spectral/retention estimates.
- `simulation/anomaly_detection.py`: run-level summary and event labels.
- `simulation/mode_diagnostics.py`: radial profiles, shape correlations, spatial distribution metrics.
- `simulation/time_resolved_diagnostics.py`: diagnostic frame capture, radial/angular diagnostics, hardened breathing detection, reports, and plots.
- `simulation/breathing_period_audit.py`: compares current diagnostic breathing periods with full-resolution metric peak periods and minimum-separated peak periods.
- `simulation/control_metrics.py`: absolute energy, decay rate, and post-cutoff core-peak metrics shared by controls.
- `simulation/artifact_controls.py`: sponge-boundary artifact controls.
- `simulation/numerical_controls.py`: smaller-dt controls.
- `simulation/grid_controls.py`: matched-proportion larger-grid controls.
- `simulation/fixed_domain_controls.py`: same-domain grid-refinement controls with best-frame resampling.
- `simulation/resolution_diagnostics.py`: fixed-domain resolution-sensitivity audits for source normalization, mask areas, energy budgets, radial profiles, and pairwise best-frame/mode-shape similarity.
- `simulation/core_modal_probe.py`: controlled direct-core modal probe orchestration, separate drive-work accounting, post-cutoff-only summaries, minimum-separated breathing checks, comparison plots, and classification.
- `simulation/transport_controls.py`: narrow source-geometry mechanism controls that compare boundary one-side/two-side/rotating variants with inner-ring, near-defect annulus, radial-peak annulus, sector, and rotating annulus drives under matched injected work.
- `simulation/prototype_3d.py`: standalone 3D prototype lattice, spherical/shell defect masks, defect-only nonlinear stiffness, six-face boundary source, cubic/random phase sources, direct core/shell controls, shell/radial diagnostics, and sponge/dt checks. It also owns 3D boundary phase offset, cubic phase sign, random phase seed, per-face amplitude scaling, coarse per-face patch phase/amplitude maps, optional second-pulse forcing, optional passive boundary-inner-edge resonator oscillator state, passive memory-mechanism stiffness profiles such as `cubic_degeneracy_split`, `radial_compensation`, `isochronous_cubic_anchor`, smooth-taper cleanup variants, passive angular high-mode damping cleanup profiles, randomized matched damping controls, and per-variant defect radius/multiplier fields used by focused controls.
- `simulation/prototype_3d_audit.py`: read-only 3D failure-mode audit that reconstructs source/sponge/defect geometry, separates near-defect shell-window energy from outer radial residue, exports radial snapshots, and classifies completed prototype runs. It reads explicit saved sponge width/strength, defect radius/multiplier, shell-defect, random-phase, and defect-nonlinearity fields when controls vary them.
- `simulation/prototype_3d_source_sponge.py`: tiny 3D source/sponge separation control that moves the boundary source to the inner sponge edge, removes source-cell sponge damping, or places the source deeper inside the domain while matching work per physical source area.
- `simulation/prototype_3d_sponge_strength.py`: tiny 3D sponge-strength control that keeps the best inner-sponge-edge source location fixed while varying weak, baseline, stronger, wider, and stronger+wider sponge settings under matched work per physical source area.
- `simulation/prototype_3d_source_geometry.py`: tiny 3D source-geometry control that compares selected boundary-face sets and phase maps under matched work per physical source area, with direct core/shell controls matched by total work.
- `simulation/prototype_3d_cubic_focus.py`: tiny 3D focused control around six-face cubic source details: deterministic repeat, cubic sign flip, global phase offset, missing face, mild face imbalance, same-coverage uniform phase, fixed-seed random repeats, and direct core/shell controls.
- `simulation/prototype_3d_cubic_confirmation.py`: tiny 3D confirmation control around original and sign-flipped cubic phases, including repeat, half-dt, weak/strong sponge, one amplitude-reduced sign-flip probe, direct core/shell controls, and 3D dt stability estimates.
- `simulation/prototype_3d_grid_confirmation.py`: tiny fixed-domain 3D grid confirmation that lifts only the clean sign-flipped cubic stronger-sponge source from 31^3 to 41^3, with optional original-cubic comparator and one negative control.
- `simulation/prototype_3d_threshold_control.py`: tiny calibrated 41^3 threshold/tolerance control around the sign-flipped cubic stronger-sponge source. It derives the target work per physical source area from the 31^3 reference, calibrates the 41^3 reference, tests explicit amplitude multipliers, work-matches phase offsets, and keeps direct core/shell controls matched by total work.
- `simulation/prototype_3d_defect_control.py`: tiny calibrated 41^3 defect-dependence control around the sign-flipped cubic stronger-sponge source. It matches every variant to the same work per physical source area, ablates defect stiffness/damping/coupling/radius, and adds fixed-window near-shell metrics anchored to the original defect radius.
- `simulation/prototype_3d_radial_window_audit.py`: tiny calibrated 41^3 current-defect vs neutral-lattice audit that scans shell windows by inner radius, exports per-window metrics, radial-profile comparisons, frame similarities, and defect-lift ratios.
- `simulation/prototype_3d_defect_lift_sweep.py`: tiny calibrated 41^3 stronger/different-defect sweep against neutral. It holds the sign-flipped cubic stronger-sponge boundary source fixed, varies only defect parameters, scans shell windows, and classifies only strict retention-plus-peak/work lift with radial-profile difference and no global outer flag.
- `simulation/prototype_3d_interference_diagnostics.py`: tiny calibrated 41^3 neutral-lattice interference diagnostic. It holds the sign-flipped cubic stronger-sponge boundary source family fixed, compares uniform/cubic-offset/random phase controls under matched work per source area, and exports phase coherence, constructive/destructive shell alignment, modal projection proxies, wavefront timing, and standing-shell persistence.
- `simulation/prototype_3d_standing_persistence.py`: tiny calibrated 41^3 settled-pattern confirmation for the two clean neutral cubic variants. It reruns sign-flipped cubic and cubic phase-offset, samples densely after a post-cutoff settling delay, and exports node/antinode stability, radial shell phase stability, shell energy autocorrelation, frame similarity, spectral concentration, and a settled standing score.
- `simulation/prototype_3d_transport_packet.py`: tiny calibrated 41^3 packet-motion audit for the two clean neutral cubic variants. It reruns sign-flipped cubic and cubic phase-offset, then exports shell-window phase velocity, radial group velocity, time-of-flight, inward/outward radial flux, shell centroid displacement, angular drift, shell exit timing, and time-lagged shell-pattern correlation.
- `simulation/prototype_3d_packet_lifecycle.py`: extended calibrated 41^3 lifecycle audit for the two clean neutral cubic variants. It preserves the cutoff but extends the run horizon, tracks packet radius, packet spread/width, shell-window peak returns, shell exit timing, inward/outward flux balance, post-cutoff shell decay, and primary/second-pulse/total positive work when active pulses are present.
- `simulation/prototype_3d_refocusing_engineering.py`: tiny calibrated 41^3 refocusing-engineering control. It reuses lifecycle diagnostics while varying only the winning engineering axes around the neutral cubic packet: phase offset, cutoff timing, fixed frequency, and optional linear chirp. It classifies whether refocus count, return-peak ratio, exit delay, decay rate, and tail retention improve without global outer-window contamination.
- `simulation/prototype_3d_refocusing_map.py`: tiny calibrated 41^3 cutoff-frequency map. It keeps the neutral cubic phase-offset source fixed, compares `cutoff_long`, `frequency_high`, their combined setting, and nearby cutoff/frequency neighbors, then classifies whether the two knobs combine constructively under strict shell-retention and outer-window guards.
- `simulation/prototype_3d_cutoff_phase_map.py`: tiny calibrated 41^3 release-phase timing map. It keeps frequency fixed, varies cutoff near 18, records phase at cutoff, compares small global phase-offset perturbations, includes a compact sign-flip/polarity family, supports passive island-refinement, sign-flip-only phase-lock needle, and threshold-robust confirmation presets, and emits ranked results by major shell-window peaks, refocus peaks, no exit, retention, outer/shell below 1.0, decay closest to zero, and global outer false. Reports include release-phase island stability, phase-lock needle-width, event-threshold sensitivity, and threshold-robust refocusing score sections.
- `simulation/prototype_3d_resonator_layer.py`: tiny calibrated 41^3 passive resonator-layer control. It keeps the sign-flip phase-lock cluster fixed, maps cutoffs 17.920-17.950, compares no-resonator, weak tuned/slightly-below/slightly-above, moderate-cubic, zero-coupling, and high-damping boundary-inner-edge auxiliary oscillators, and writes threshold-robust scores plus separate resonator energy and coupling-exchange timeseries. The resonator has no external drive; pass/fail interpretation depends on strict-count width, decay, outer/shell, post-cutoff external work, and energy-accounting consistency.
- `simulation/prototype_3d_release_phase_return_map.py`: read-only analysis for completed 3D refocusing artifacts. It loads summary, ranked, threshold-robust, event, and timeseries CSVs where available; repairs CSV boolean controls when robust rows must be recomputed; builds the release-phase feature table; emits phase-binned, nearest-neighbor, linear, and logistic predictor outputs; and writes the `predictive phase rule` plus `blind confirmation recommendation` report sections.
- `simulation/prototype_3d_release_phase_blind_confirmation.py`: fixed passive blind confirmation for the return-map predictor. It reuses the calibrated sign-flip lifecycle runner and threshold-robust scoring, assigns prediction roles to the five pre-registered cutoffs, writes summary/prediction-check CSVs, and classifies whether predicted strong rows beat the weak control under strict clean 9/8 criteria.
- `simulation/prototype_3d_release_phase_numerical_validation.py`: fixed passive numerical validation for the blind-confirmed phase rule. It reuses the same calibrated sign-flip lifecycle runner, repeats the pre-registered strong/lower-side cutoffs at baseline and half dt, optionally adds quarter dt only when requested, writes summary/comparison CSVs, and classifies whether the phase ordering is numerically confirmed, dt-sensitive, inconclusive, or failed.
- `simulation/prototype_3d_release_phase_dt_recenter.py`: fixed passive half-dt recentering control. It reuses the same calibrated sign-flip lifecycle runner and threshold-robust scoring, runs only the preselected recenter cutoffs plus low-side controls at half dt, writes summary and threshold-robust CSVs, and classifies whether strict 9/8 recenters, collapses to a single row, fails, or remains threshold-free/inconclusive.
- `simulation/prototype_3d_release_phase_proof_pack.py`: fixed passive quarter-dt proof pack. It reuses the calibrated sign-flip lifecycle runner and threshold-robust scoring, runs only the recentered proof candidates plus immediate/low-side controls, writes the frozen `candidate_card.md`, summary, threshold-robust, and gates CSVs, and classifies whether neighboring strict 9/8 plus threshold-free lifecycle gates support the phase rule.
- `simulation/prototype_3d_release_phase_resolution_lift.py`: one-step passive resolution-lift control. It inherits the proof-pack setup, recomputes cutoffs from target release phases instead of copying lower-resolution times, defaults to `51^3` and quarter dt, compares a phase-pocket candidate with low-side and weak negative controls, and writes report, summary, threshold-robust, gates, and JSON artifacts.
- `simulation/prototype_3d_release_phase_resolution_postmortem.py`: read-only postmortem for completed proof-pack and resolution-lift artifacts. It loads summary/robust/event/timeseries data, compares proof winners against the lifted candidate and controls, separates missing returns from below-gate return shrinkage, inspects radial/timing shifts, and emits a `no_recalibrated_retry` or single predicted retry recommendation without running physics.
- `simulation/prototype_3d_release_phase_modal_audit.py`: read-only modal audit for completed proof-pack, resolution-lift, postmortem, and central-burst artifacts. It diagnoses dominant shell-energy frequency, spectral concentration and bandwidth, autocorrelation decay, return timing jitter, peak widths, radial group velocity, radial packet spread, scalar shell-energy phase locking, neighboring radial-window energy proxies, and central-burst contrast rows, then classifies scale failure as blur, finite-grid resonance, scalable modal rule, or inconclusive without running physics.
- `simulation/prototype_3d_release_phase_dispersion_audit.py`: read-only dispersion/blur model for completed proof-pack, resolution-lift, and postmortem artifacts. It reconstructs deterministic 41^3/51^3 source masks and shell-window sampling from the baseline config without running physics, compares return bandwidth, tail-radius drift, peak spacing/width, radial packet spread, shell leakage, source discretization, shell scaling, and spatial-phase-frame availability, then classifies whether a scalable blur model, source correction, shell-window correction, finite-grid resonance, or no safe next candidate is supported.
- `simulation/prototype_3d_spatial_phase_instrumentation.py`: instrumentation-only two-row reproduction for the passive release-phase scale question. It reruns the fixed `41^3` proof row and failed `51^3` lift candidate with the frozen neutral/sign-flip setup, stores shell displacement/velocity frames at loose return peaks, radial shell phase frames, shell coherence by radius, angular/spherical coherence, node/antinode stability maps, phase drift across returns, and a `41^3` vs `51^3` comparison. It classifies whether scale loss looks like coherent blur, spatial decoherence, shell-window alignment, or unresolved finite-resolution behavior.
- `simulation/prototype_3d_smooth_envelope_resolution_lift.py`: fixed `51^3` temporal source-spectrum test. It reuses the spatial phase instrumentation lifecycle path, runs hard-control/smooth-candidate/smooth-negative rows, suppresses raw node-frame row expansion for this heavy control while retaining shell/radial/angular coherence summaries, checks source bandwidth reduction, count/coherence gates, tail-radius shift, and clean work/no-exit/global-outer guards, and classifies smooth-envelope rescue, partial coherence-only improvement, count-without-coherence suspicion, or no rescue.
- `simulation/prototype_3d_boundary_phase_conjugate.py`: fixed patch-level boundary phase-conjugate control. It captures the 41^3 proof row shell displacement/velocity frames, derives a coarse 96-patch boundary phase/amplitude mask from the first proof returns, freezes `boundary_phase_conjugate_candidate.json` before any 51^3 row, runs hard/candidate/shuffled/amplitude-only/phase-only/wrong-return controls with the same release phase/work/source geometry, and classifies only when strict counts and shell/radial/angular coherence improve together while shuffled patches fail.
- `simulation/prototype_3d_modal_sparsity_audit.py`: read-only sparse modal audit. It loads existing lifecycle time series/events plus threshold-robust summaries, computes FFT-based top-mode reconstruction counts for 90/95/99%, modal participation ratio, return beat-period regularity, half-height peak widths, and peak-width/modal-density correlations, then classifies whether source-shaped `51^3` controls share one signature.
- `simulation/prototype_3d_return_family_gate_audit.py`: read-only return-family gate audit. It loads the same completed artifacts plus modal-sparsity output, builds return windows from early shell peaks, computes comb occupancy/score, threshold crossing tables, indexed return strength, local prominence, on-comb/off-comb energy, late-return survival, and plots, then classifies whether strict `51^3` loss is a gate artifact, real family weakening, inconclusive, or missing artifacts.
- `simulation/prototype_3d_off_comb_leakage_audit.py`: read-only off-comb leakage localization. It consumes the return-family gate audit plus proof/lift/spatial/smooth/phase-conjugate/modal artifacts, attaches saved lifecycle traces where available, computes radial-window leakage proxies, angular-sector coherence, lagged outer/off-comb correlation, modal sideband fractions, spatial-pattern leakage, and flux summaries, then classifies the dominant leakage channel without running physics.
- `simulation/prototype_3d_return_pattern_symmetry_audit.py`: read-only return-pattern symmetry audit. It consumes saved off-comb rows plus node-level shell displacement/velocity frames or angular-sector shell patterns where available, computes raw/sign/phase/cubic/reflection/sector-shift/penalized-permutation/harmonic return-pair similarities, transform stability, and source-control transform signature sharing, then classifies recoverable drift, hopping, phase precession, true scrambling, inconclusive, or missing artifacts without running physics.
- `simulation/prototype_3d_branch_closure_report.py`: read-only passive branch closure synthesis. It consumes saved proof/lift/postmortem/modal/dispersion/spatial/source-shape/phase-conjugate/modal-sparsity/gate/leakage/symmetry/resonator/central-burst/second-pulse artifacts, writes summary/evidence/claim/forbidden/reopen CSVs plus Markdown/JSON, and classifies whether the passive scale-lift branch is closed, incomplete, or missing artifacts.
- `simulation/prototype_3d_spatial_memory_mechanism_lab.py`: independent passive mechanism lab for the reopen condition. It constructs neutral, anisotropy-anchor, cubic-splitting, shell-band-isolation, nonlinear-phase-memory, and randomized equivalent-strength variants; records shell return frames; scores return-to-return spatial-pattern memory plus comb/off-comb/coherence/modal/gate metrics; and classifies only against neutral and randomized controls. It reuses the frozen neutral sign-flip lifecycle setup while adding weak passive lattice-memory profiles only; guardrails reject cutoff retuning, `61^3`, source-shape variants, and closed-branch rescue framing.
- `simulation/prototype_3d_cubic_memory_tradeoff_map.py`: fixed 41^3 cubic degeneracy-splitting tradeoff map. It builds standard/sign-flipped cubic split strength rows plus matched randomized controls, reuses the spatial-memory frame metric and lifecycle gates, writes cubic-memory summary/by-return/comparison/report/plot artifacts, and classifies strict-count-preserving support, memory-only tradeoff, no tradeoff, or invalid tradeoff.
- `simulation/prototype_3d_cubic_memory_survivor_bias_audit.py`: read-only survivor-bias audit. It loads saved cubic-memory summary and shell displacement frames, remaps frames onto neutral-predicted return windows, computes surviving-peak, equal first-N, neutral-window, by-index, late-decay, strict-count, and comb-score relationships, then classifies real same-window gain, survivor bias, inconclusive, or missing artifacts.
- `simulation/prototype_3d_isochronous_cubic_memory_anchor.py`: fixed 41^3 isochronous cubic-memory anchor test. It constructs neutral, cubic split, radial compensation, isochronous anchor, and matched randomized rows; reuses the spatial-memory instrumentation path; writes summary/by-return/comparison/report/plot artifacts; and classifies supported, memory-only tradeoff, no signal, or invalid. ## Fixed-Domain Semantics

Fixed-domain mode is opt-in with `fixed_domain: true`.

When enabled:

- `dx = domain_width / (nx - 1)`
- `dy = domain_height / (ny - 1)`
- Defect radius can be specified as `defect.radius_physical`.
- Core radius can be specified as `core_radius_physical`.
- Sponge width can be specified as `boundary_damping_width_physical`.
- Emitter strip width can be specified as `driver.emitter_width_physical`.
- Fixed-domain emitters support fractional source coverage weights for physically comparable source geometry.
- `driver.source_normalization` supports `per_cell`, `per_length`, `constant_boundary_flux`, and `constant_total_work`.
- `drive_location` supports `boundary`, `core_node`, `core_region`, and `annulus`.
- Direct core drives support `burst`, `impulse`, `chirp`, and `continuous` modes with separate work accounting from boundary drives.
- Direct annulus/core-region drives support physical inner/outer annulus radii, optional angular sectors, and uniform or rotating spatial phase maps.
- Lattice coupling force scales by `1/dx^2` and `1/dy^2`.
- Energy density integrates onsite/kinetic/nonlinear/coupling contributions with cell area.
- Masks, sponge damping, phase maps, radial profiles, annulus masks, and angular masks use physical distances.
- Time-resolved radial diagnostics report physical radii.

Historical non-fixed-domain configs still work and retain cell-unit semantics.
