# WaveEngine Roadmap

This file is the project roadmap and should be updated whenever we complete a meaningful task, change direction, add a new phase, or discover a better next step.

## Current Next Step

The 3D branch is now structured boundary-interference transport and refocusing, not a defect well or confirmed standing-shell program. Source timing/frequency can improve repeated shell-window returns, the cutoff-frequency map showed the knobs are not simply additive, the tighter cutoff/polarity release-phase map found a supported timing island, and ultra-fine passive refinement found a narrow phase-lock needle rather than a broad timing island.

Current state: the passive scale-lift branch is protected and archived unless a genuinely new stable spatial-pattern-memory mechanism appears. In `runs\release_phase_proof_pack_3d_20260619_234039`, the fixed quarter-dt `41^3` proof pack classified as `release_phase_quarter_dt_proof_supported`: proof-candidate cutoffs `17.94`, `17.9425`, and `17.945` preserved strict clean 9/8, with phase `0.5048-0.5094`. The requested one-step recalibrated resolution lift then ran at `51^3` in `runs\release_phase_resolution_lift_3d_20260620_091834` and classified as `release_phase_resolution_lift_failed`; the read-only postmortem in `runs\release_phase_resolution_postmortem_3d_20260620_100043` classified as `resolution_lift_blurred_returns_no_predictive_recalibration`. The modal/dispersion/spatial-phase audits showed the same dominant shell-energy band near `0.012807`, but `51^3` loses spatial phase coherence and strict refocusing. The low-dimensional phase-precompensation design in `runs\spatial_phase_precomp_design_3d_20260620_175852` classified as `no_safe_phase_correction`. The read-only source-spectrum audit in `runs\source_spectrum_design_audit_3d_20260620_181010` justified one final same-release temporal smoothing test, which ran in `runs\smooth_envelope_resolution_lift_3d_20260620_192501` and classified as `smooth_envelope_no_rescue`: the smooth `sin^2` source reduced sidebands as predicted, but default/strict counts fell to 9/8 and 7/6, shell/radial/angular coherence all worsened versus the hard `51^3` control, and tail-radius shift worsened despite clean no-exit/global-outer/work gates. The measured patch-level boundary phase-conjugate mirror in `runs\boundary_phase_conjugate_3d_20260620_212918` also classified as `boundary_phase_conjugate_no_rescue`: the 96-patch candidate stayed default 9/8, strict 7/6, and loose 11/10, did not improve shell/radial/angular coherence versus the hard `51^3` control, and the shuffled patch control did not fail. The read-only modal sparsity audit in `runs\modal_sparsity_audit_3d_20260620_231602` classified as `common_51_source_signature_supported`: it did not prove a dramatic few-mode-to-20-plus-mode transition, but hard/smooth/phase-conjugate/shuffled `51^3` rows shared a tight reconstruction/participation signature and the same strict-count loss. The read-only return-family gate audit in `runs\return_family_gate_audit_3d_20260621_082543` classified as `return_family_weakened_not_gate_artifact`: timing and comb occupancy stay organized, but off-comb energy is high and amplitude/prominence compression does not explain the strict-count loss. The read-only off-comb leakage audit in `runs\off_comb_leakage_audit_3d_20260621_085347` classified as `spatial_pattern_scrambling_supported`: radial leakage, modal sidebands, and delayed outer recycling did not separate from proof rows, while source-control spatial-pattern leakage rose from `0.495788` to `0.586679`. The read-only return-pattern symmetry audit in `runs\return_pattern_symmetry_audit_3d_20260621_091511` classified as `pattern_symmetry_inconclusive`: alignment can raise `51^3` pattern memory, but transform stability is low (`0.313696` source-control mean) and source controls do not share one transform signature (`0.363636` share), so orientation drift, cubic-mode hopping, and phase precession are not supported as recoverable mechanisms. The closure packet in `runs\branch_closure_report_3d_20260621_093821` classified as `passive_scale_lift_branch_closed`: freeze the `41^3` structured-refocusing claim, freeze the no-`51^3`-scale-validation non-claim, and treat the `51^3` failure as organized timing with lost spatial-pattern identity/return-family purity. The first central high-frequency burst ladder in `runs\central_burst_3d_20260620_103248` classified as `central_burst_transient`. Do not run a precompensated `51^3` candidate, the proposed old 51^3 retry, `61^3`, nearby cutoff tuning, central-burst broadening, source-shape sweeps, modal-sparsity-derived source variants, return-gate detector tuning, off-comb-derived source masks, symmetry/orientation-drift follow-ups, or phase-conjugate patch-mask tuning. The current conservative conclusion is a well-characterized `41^3` passive phase-lock proof, not a scale-validated packet-control law.

New independent research branch: `spatial_memory_mechanism_lab`. This branch asks whether any passive lattice mechanism can preserve return-to-return shell spatial-pattern identity. It is not a continuation or rescue of the closed scale-lift branch. The first fixed run in `runs\spatial_memory_mechanism_lab_3d_20260621_103028` classified as `spatial_memory_mechanism_supported`: at `41^3`, cubic degeneracy splitting produced the best memory score (`0.645969`) and beat both the neutral reference (`0.486969`) and randomized equivalent-strength control (`0.505821`) under clean gates. The gated optional `51^3` follow-up did not preserve that advantage: cubic degeneracy splitting scored `0.544617` versus neutral `0.579914` and randomized control `0.577519`, although it modestly improved strict count to `8/7`. Treat this as a genuine 41^3 passive spatial-memory mechanism clue, not as scale validation or a reopened old 51^3 rescue path.

The mechanism-specific local tradeoff map in `runs\cubic_memory_tradeoff_map_3d_20260621_142657` classified as `memory_only_tradeoff_supported`. Cubic degeneracy splitting still gives a real 41^3 spatial-memory signal, but the tested local strength/orientation settings did not preserve the strict `9/8` floor. The best memory row was sign-flipped cubic split `0.5x` (`0.725354` memory versus neutral `0.486969` and randomized `0.504878`) but dropped to strict/default/loose `6/5`, `7/6`, `8/7`. The standard `1.0x` cubic split stayed closer to the original mechanism clue (`0.645969` versus matched randomized `0.628214`) but remained strict/default/loose `8/7`, `9/8`, `10/9`. All gates and work accounting passed, so this is a clean memory/count tradeoff rather than an invalid run. No cubic row achieved `cubic_memory_tradeoff_supported`.

The read-only survivor-bias audit in `runs\cubic_memory_survivor_bias_audit_3d_20260621_150538` classified as `cubic_memory_tradeoff_inconclusive`. Standard cubic rows retain some matched neutral-window memory gain, but the largest sign-flipped memory gains coincide with low neutral-window coverage. The audit therefore does not support a clean claim that cubic memory is purely same-window improvement, nor a clean claim that it is entirely survivor bias.

Next fixed mechanism question, now implemented but not yet run: `prototype-3d-isochronous-cubic-memory-anchor` tests whether a weak cubic degeneracy split plus a smooth radial compensation profile can keep the spatial-memory signal while preserving strict `9/8` and near-neutral comb score. It is a small `41^3`-only map: neutral, cubic split `0.5x/1.0x`, radial compensation only, isochronous anchor `0.5x/1.0x`, and randomized matched controls `0.5x/1.0x`. It must not run `51^3` by default and must not tune cutoff, source shape, active pulses, resonators, or `61^3`.
